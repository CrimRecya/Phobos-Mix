#### 方向装甲

  > - *pr #1623 by NetsuNegi*
  > - [返回目录](#额外功能目录)

在 `rulesmd.ini` 之中：

```ini
[CombatDamage]                               ; 全局伤害设置
DirectionalArmor=no                          ; 布尔值，是否启用方向护甲逻辑
DirectionalArmor.FrontMultiplier=1.0         ; 浮点型，正面受击时，伤害倍率
DirectionalArmor.SideMultiplier=1.0          ; 浮点型，侧面受击时，伤害倍率
DirectionalArmor.BackMultiplier=1.0          ; 浮点型，背面受击时，伤害倍率
DirectionalArmor.FrontField=0.5              ; 浮点型，正面判定范围
DirectionalArmor.BackField=0.5               ; 浮点型，背面判定范围

[SOMEVEHICLE]                                ; 任意车辆
DirectionalArmor=                            ; 布尔值，是否启用方向护甲逻辑，默认值为 `CombatDamage` 中 `DirectionalArmor` 的值
DirectionalArmor.FrontMultiplier=            ; 浮点型，正面受击时，伤害倍率，默认值为 `CombatDamage` 中 `DirectionalArmor.FrontMultiplier` 的值
DirectionalArmor.SideMultiplier=             ; 浮点型，侧面受击时，伤害倍率，默认值为 `CombatDamage` 中 `DirectionalArmor.SideMultiplier` 的值
DirectionalArmor.BackMultiplier=             ; 浮点型，背面受击时，伤害倍率，默认值为 `CombatDamage` 中 `DirectionalArmor.BackMultiplier` 的值
DirectionalArmor.FrontField=                 ; 浮点型，正面判定范围，默认值为 `CombatDamage` 中 `DirectionalArmor.FrontField` 的值
DirectionalArmor.BackField=                  ; 浮点型，背面判定范围，默认值为 `CombatDamage` 中 `DirectionalArmor.BackField` 的值

[SOMEWARHEAD]                                ; 任意弹头
Directional=no                               ; 布尔值，弹头是否具有方向性，具有方向性的抛射体击中具有方向性的载具或载具的护盾时将会触发方向护甲判定
Directional.Multiplier=1.0                   ; 浮点型，方向护甲影响倍率
```

```{note}
- 判定范围始终都是对称的，正面 0.5 表示以车头方向为轴，左右各 45 度角范围内均属于正面判断范围，1.0 则表示左右各 90 度均为正面判定范围
- 侧面范围是除开正面和背面后剩余的角度范围
```

#### 残骸系统

  > - 暂无pr *by TaranDahl*
  > - [返回目录](#额外功能目录)

在 `rulesmd.ini` 之中：

```ini
[General]                                    ; 全局通用设置
EnableWreckageSpawn=no                       ; 布尔值，是否开启以下的残骸相关功能
WreckageInitialHealthPercent=0.1             ; 浮点型，残骸初始生命比例

[SOMETECHNO]                                 ; 任意步兵、车辆、飞机或建筑
WreckageType=                                ; Techno列表，残骸的类型，填多个会随机生成其中一个
WreckageInitialHealthPercent=                ; 浮点型，残骸初始生命比例，默认值为 `General` 中 `WreckageInitialHealthPercent` 的值
WreckageDeactive=yes                         ; 布尔值，残骸是否会被断电，就像遥控坦克失去控制中心了一样
WreckageMarkUp=yes                           ; 布尔值，残骸是否不会产生碰撞体积，不会被各种范围查找效果找到，只有精控、超时空等点名该单位的效果才能对其造成影响
WreckageOwner=Victim                         ; 归属类型，Invoker/Killer/Victim/Civilian/Special/Neutral（Invoker和Killer都是造成伤害的作战方，Victim是死亡单位的作战方）
WreckageLeaveOnWater=no                      ; 布尔值，单位位于水面时是否生成残骸
WreckageLeaveInAir=no                        ; 布尔值，单位位于空中时是否生成残骸
WreckageSwapLocomotor=no                     ; 布尔值，单位是否会跟残骸交换 Locomotor ，这有助于残骸保持在正确的位置上，但请确保原有的单位在受伤后会立即死亡

[SOMEWARHEAD]                                ; 任意弹头
SuppressWreckage=no                          ; 布尔值，被该弹头击杀的单位是否不会留下残骸
ActivateWreckage=no                          ; 布尔值，被该弹头伤害的残骸是否会恢复行动能力
```

```{note}
- 残骸的类别（步兵、车辆、飞机或建筑）不必和原本单位一致，新的单位会被强制放置在单位原本的位置，你可以使用 `WreckageMarkUp` 避免残骸与大多数效果产生交互，但如果你让残骸移动，那么 `WreckageMarkUp` 的效果会失效，`WreckageMarkUp` 建议和禁止选择、鼠标忽视物件等功能配合使用
- 如果残骸的类型是自己，那么当单位已经是残骸时，将不再生成残骸
- 单位必须有一个作战方，如果定义的归属作战方无效，那么就会使用死亡单位的作战方
- 建筑残骸目前没有额外支持（Ares已有废墟）
```

#### 多炮管和多炮塔

  > - 暂无pr *by TaranDahl & CrimRecya*
  > - [返回目录](#额外功能目录)

在 `artmd.ini` 之中：

```ini
[SOMEVEHICLE]                                ; 任意 VXL 车辆
BarrelOverTurret=                            ; 布尔值，炮管图层是否永远在炮塔之上，不填为原本的按照方向自动计算
BarrelOffset=0                               ; 整数型，主炮管相对炮塔中心点在Y轴（左右）方向上的偏移
ExtraBarrelCount=0                           ; 整数型，每个炮塔额外绘制的炮管数量
ExtraBarrelOffsetX=0                         ; 整数型，额外绘制的炮管相对炮塔中心点在Y轴（左右）方向上的偏移，X 是额外炮管索引，从 0 开始
ExtraTurretCount=0                           ; 整数型，额外绘制的炮塔数量
ExtraTurretOffsetX=0,0,0                     ; FLH坐标，请写完整，额外绘制的炮塔相对单位中心点的偏移，X 是额外炮塔索引，从 0 开始
BurstPerTurret=0                             ; 整数型，每个炮塔 Burst 多少次后会切换到下一个炮塔
```

```{note}
- 现在你可以让载具将自己的炮塔在多个位置进行绘制并开火，仅支持常规vxl载具
P.S.1: 单位开火时会首先使用原本的炮塔开火 `BurstPerTurret` 次，然后切换到0号额外炮塔开火 `BurstPerTurret` ，然后切换到 1 号，依此类推。开火时的 FLH 也是相对开火时炮塔的位置计算的，若单位的炮塔或炮管设置了制退效果，则会根据 `Burst` 自动计算炮塔炮管编号
P.S.2: 开火位置本质上是由当前的 `Burst` 决定的，如果你要让后面的炮塔开火，就得有 `Burst` 很高的武器才行。例如一个具有 2 个额外炮塔（共 3 个炮塔）的战列舰，`BurstPerTurret=3` ，那么需要一个至少 `Burst=9` 才能让所有炮塔完整开火一遍，炮管在每个炮塔上同理
P.S.3: 这本质上是为了复刻红警1的多炮塔，而不是帮助你在红警2里制作赞托斯，所以不同炮塔使用不同武器/分别索敌/分别计算冷却/分别具有生命值等都不在这个功能的考虑范围内，相关需求请使用替身/附加单位
```

```ini
Turret.BodyOrientationSymmetric=yes          ; 布尔值，身体对准目标时额外的转动角度是否左右均可，不必判断具体往哪一边，即身体既可以用左侧也可以用右侧对着目标
```

```{note}
- 最终效果：
  - 当你建造飞机 A 时，机场会刷出一个 B 来；建造时间、造价等都是按 A 的来，而刷出位置、被克隆等都是按 B 的来
```

#### 高级驾驶移动方式

  > - *pr #1622 by CrimRecya*
  > - [返回目录](#额外功能目录)

在 `rulesmd.ini` 之中：

```ini
[SOMEVEHICLE]                                ; 只能是车辆
Locomotor=AdvancedDrive                      ; 移动方式，以下功能仅在此移动方式下可用 {4A582751-9839-11d1-B709-00A024DDAFD1}
AdvancedDrive.Reverse=yes                    ; 布尔值，单位是否会倒车，启用后能够使用下面的五个选项
AdvancedDrive.Reverse.FaceTarget=yes         ; 布尔值，当单位指定了目标时，在上述距离内强制正面向敌进行移动，不再自动选择
AdvancedDrive.Reverse.FaceTargetRange=16.0   ; 浮点型，当单位指定了目标时，离目标距离多远内，会倒车来保持正面相对
AdvancedDrive.Reverse.MinimumDistance=2.5    ; 浮点型，当单位制定了目的地时，离目的地距离多远内，会自动选择倒车移动
AdvancedDrive.Reverse.RetreatDuration=150    ; 整数型，当单位制定了目的地时，距上次受到伤害这么长时间内，会倒车前往目的地
AdvancedDrive.Reverse.Speed=0.85             ; 浮点型，倒车时的速度倍率
AdvancedDrive.Hover=no                       ; 布尔值，单位是否会悬浮，启用后能够使用下面的六个选项
AdvancedDrive.Hover.Sink=yes                 ; 布尔值，单位是否会在水面上被动失去动力时沉没
AdvancedDrive.Hover.Spin=yes                 ; 布尔值，单位是否会在被动失去动力时旋转
AdvancedDrive.Hover.Tilt=yes                 ; 布尔值，单位是否会在斜坡上倾斜
AdvancedDrive.Hover.Height=                  ; 整数型，单位悬浮时的标准高度，默认值为 `General` 中 `HoverHeight` 的值
AdvancedDrive.Hover.Dampen=                  ; 浮点型，单位悬浮时的弹性程度，默认值为 `General` 中 `HoverDampen` 的值
AdvancedDrive.Hover.Bob=                     ; 浮点型，单位悬浮时的浮动间隔，默认值为 `General` 中 `HoverBob` 的值
```

```{note}
- 一般情况下，都会正面朝向移动方向
- 所有使用限制和 `Locomotor=Drive` 相同
- 同时，`Locomotor=AdvancedDrive` 相比 `Locomotor=Drive` 修复了在进行日字形移动时不会碾压到中间格的问题，且上下坡转动将更为顺滑
- 与 `DirectionalArmor` 和 `ExpandTurretRotation` 相性较佳，与 `KeepTargetOnMove` 和 `KeepRange` 同时使用则能提升操作手感
```

```ini
ProximityAllies=no                           ; 布尔值，撞击伤害是否对盟友生效
```

```ini

DisperseBursts=                              ; 整数型列表，抛体每次各扩散几个对应的武器，如果数量少于 `DisperseWeapons` ，将使用该列表的最后一项
DisperseCounts=                              ; 整数型列表，抛体需要各扩散几次对应武器，为零会直接计算冷却而不发射，负数视为无限，如果数量少于 `DisperseWeapons` ，将使用该列表的最后一项
DisperseDelays=                              ; 整数型列表，抛体在发射对应次数后需要等待多久才能进行下一次发射，至少为1，如果数量少于 `DisperseWeapons` ，将使用该列表的最后一项
DisperseCycle=0                              ; 整数型，抛体能够发射几轮武器，为零不会发射武器，负数视为无限
DisperseInitialDelay=0                       ; 整数型，抛体在发射后多少帧第一次扩散武器，即扩散武器的初始延迟
DisperseEffectiveRange=0                     ; 浮点型，抛体扩散武器计时器会在距离目标这么近的距离时才会开始计时，为0代表不启用该逻辑，为负数代表只有在即将引爆本体的瞬间扩散最多一次武器
DisperseSeparate=no                          ; 布尔值，抛体每次扩散武器时，是否不再一次性发射列表中的所有武器，改为按列表顺序发射列表中的一组武器，遵循其 `DisperseBurst` ，即：
;                                            ; 是则，按顺序发射一组 `DisperseWeapons` ，发射这组武器对应 `DisperseBursts` 中的数量，等待的冷却为这组武器对应 `DisperseDelays` 中的时间，
;                                            ; 在发射这组武器对应的 `DisperseCounts` 中的次数后，切换到下一组 `DisperseWeapons` 并继续；
;                                            ; 否则，发射所有 `DisperseWeapons` ，每种武器发射该武器对应 `DisperseBursts` 中的数量，等待的冷却为此时次数对应 `DisperseDelays` 中的时间，
;                                            ; 在发射此时次数对应的 `DisperseCounts` 中的次数后，切换到下一组 `DisperseCounts` 并继续，两者均会重复 `DisperseCycle` 中的轮次数
DisperseRetarget=no                          ; 布尔值，抛体每次扩散武器时，扩散的武器是否自行寻找新目标，寻找目标的范围使用武器的 `Range`, 会读取 `DisperseWeapons` 的 `CanTarget` 并判断攻击目标所需要的 `AttachedEffects` 是否满足
DisperseLocation=no                          ; 布尔值，抛体每次扩散武器时，扩散的武器在自行寻找新目标时，是否以扩散位置为中心的位置寻找，否则以原目标为中心位置寻找
DisperseTendency=no                          ; 布尔值，抛体每次扩散武器时，扩散的武器中每组的第一个武器是否必定会选择原目标
DisperseHolistic=no                          ; 布尔值，抛体每次扩散武器时，扩散的武器是否选择和原目标不同状态（在空中和在地上两种状态）的目标
DisperseMarginal=no                          ; 布尔值，抛体每次扩散武器时，扩散的武器是否会选择数木、路灯或可拦截抛体等物体作为次要目标
DisperseDoRepeat=no                          ; 布尔值，抛体每次扩散武器时，扩散的武器是否会因为搜索到的目标数量不足而尝试攻击相同目标，即：
;                                            ; 是则，当武器可以攻击单位和地面时，优先攻击单位，然后开始尝试重复攻击单位，只有当本轮没有任何可被攻击的单位存在于范围内时才会尝试攻击地面；
;                                            ; 否则，当武器可以攻击单位和地面时，优先攻击单位，然后尝试攻击所有可被攻击的地面格子，若本轮所有可选目标都已被锁定而仍能发射，则会停止发射剩余武器
DisperseSuicide=yes                          ; 布尔值，抛体本体是否在扩散武器次数耗尽后自爆，原本没有次数时无效
DisperseFromFirer=                           ; 布尔值，抛体发射的扩散武器是否由单位朝抛体发射，否则是由抛体朝目标发射，不同弹道类型使用不同默认值
;                                            ; Straight - 否
;                                            ; Bombard - 否
;                                            ; Missile - 否
;                                            ; Engrave - 是
;                                            ; Parabola - 否
;                                            ; Tracing - 是
DisperseFaceCheck=no                         ; 布尔值，抛体在发射武器前是否检查图像朝向，如果 `Trajectory.BulletFacing=Velocity` 或 `Trajectory.BulletFacing=Spin` 时无视该项强制不检查
DisperseForceFire=yes                        ; 布尔值，抛体本身在没有目标或目标超出射程时是否仍然发射扩散武器
DisperseCoord=0,0,0                          ; FLH坐标，请写完整，从抛体位置发射的武器的具体位置会叠加该位置，可以视作是抛体发射武器的FLH（即需要 `DisperseFromFirer=no`）
```

```{warning}
- 该功能为面向 [弹道抛体系统](#弹道抛体系统) 设计，如在原版抛体中使用时出现问题，请替换为新的弹道抛体
```

```{note}
- 注意 `DisperseWeapons` 中使用的武器必须注册，且尽管支持嵌套使用，但请合理运用
- 如果你设置了 `DisperseRetarget=yes` ，那么同样的，请给 `DisperseWeapons` 中所列出的所有武器的 `Range` 设置一个比较小的值，过大的值会显著增加性能消耗
- 尽管 `DisperseDoRepeat=no` 会禁用重复选择目标，但若武器能够攻击地面，则仍然可能因为锁定到目标所处的格子而攻击到重复目标
```

```ini
[SOMEPROJECTILE]                             ; 任意抛体
LifeDuration=0                               ; 整数型，抛体能够存在的时间，单位为帧，不同弹道类型存在不同含义
;                                            ; Straight - 抛体能够存在的时间，时间结束时抛体会引爆，如果为非正数，则不计时
;                                            ; Bombard - 抛体能够存在的时间，时间结束时抛体会引爆，如果为非正数，则不计时
;                                            ; Missile - 抛体能够存在的时间，时间结束时抛体会引爆，如果为非正数，则不计时
;                                            ; Engrave - 切割过程的持续时间，为非正数则会根据 `Trajectory.Engrave.SourceCoord` 和 `Trajectory.Engrave.TargetCoord` 自动计算时间，此时 `Trajectory.Engrave.TargetCoord` 可视作切割线段的终点坐标
;                                            ; Parabola - 抛体能够存在的时间，时间结束时抛体会引爆，如果为非正数，则不计时
;                                            ; Tracing - 抛体锁定的持续时间，时间结束时抛体会引爆，如果为零，则使用武器 `ROF` - 10 作为持续时间，持续时间最低为1，如果为负数，则不计时
NoTargetLifeTime=-1                          ; 整数型，抛体失去目标闲置时的容忍时间，抛体在失去目标超过这么长时间后会引爆，若为0，则会在切换目标时直接引爆
CreateCapacity=-1                            ; 整数型，设置为非负数时，只有当发射者发射的弹道抛体在地图上的数量小于该值时才能发射该弹道抛体
ApplyRangeModifiers=no                       ; 布尔值，抛体是否会受到射程AE的影响，抛体 `LifeDuration` 、`Trajectory.DetonationDistance` 和 `Trajectory.EdgeAttenuation` 的计算是否会受到射程AE的影响
PeacefulVanish=                              ; 布尔值，抛体在最后引爆时是否直接消失，没有动画和伤害，默认值为 `Trajectory=Engrave` 或 `ProximityImpact` 不等于0 或 `DisperseCycle` 不等于0
```


#### 弹道抛体系统

  > - *pr #1582 by CrimRecya*

在 `rulesmd.ini` 之中：

```ini
[SOMEPROJECTILE]                             ; 弹道抛体（不和原版四类抛体共存）
; ------------------------------------------ ; 弹道抛体的弹道类型
Trajectory=                                  ; Trajectory类型，包括 Straight/Bombard/Missile/Engrave/Parabola/Tracing
; ------------------------------------------ ; 弹道抛体的通用基本功能
Trajectory.Speed=100.0                       ; 浮点型，弹道抛体的速度，不同弹道类型存在不同含义
;                                            ; Straight - 抛体的速度
;                                            ; Bombard - 抛体在上升阶段的速度
;                                            ; Missile - 抛体在 `Trajectory.Missile.UniqueCurve=no` 的情况下时的最终速度
;                                            ; Engrave - 抛体在切割过程的水平移动速度（最高速度限制在128）
;                                            ; Parabola - 抛体在水平方向上的速度，且仅用于模式0、模式3和模式5
;                                            ; Tracing - 抛体在追击目标时的速度
Trajectory.BulletROT=0                       ; 整数型，抛体图像的转动速度，不影响移动方向，为0时会始终朝向 `Trajectory.BulletFacing` 中的方向，分别有不同的作用
Trajectory.BulletFacing=Velocity             ; 面向模式枚举类，抛体的面向模式，包括 Velocity/Spin/Stable/Target/Destination/FirerBody/FirerTurret
Trajectory.BulletFacingOnPlane=no            ; 布尔值，抛体的面向是否只会处在水平面上
Trajectory.OffsetCoord=0,0,0                 ; FLH坐标，请写完整，抛体目标位置的额外偏移，抛体会瞄准目标的这个相对坐标进行攻击，支持在此基础上启用随机散布
;                                            ; Straight - 有效
;                                            ; Bombard - 有效
;                                            ; Missile - 有效
;                                            ; Parabola - 有效
Trajectory.RotateCoord=0                     ; 浮点型，是否按照武器的 `Burst` 在以 `Trajectory.OffsetCoord` 为角平分线的角度内旋转抛体发射方向，为0代表不启用该逻辑（负值会反转旋转方向）
;                                            ; Straight - 有效
;                                            ; Bombard - 有效
;                                            ; Missile - 有效，但 `Trajectory.OffsetCoord` -> `Trajectory.Missile.PreAimCoord`
;                                            ; Parabola - 有效
Trajectory.MirrorCoord=yes                   ; 布尔值，是否镜像坐标，如果发射者开火时当前武器的 `Burst` 数为偶数，则 `Trajectory.OffsetCoord` 中的L会镜像翻转，同时 `Trajectory.RotateCoord` 计算的旋转方向也会反转，每侧相邻抛体间的旋转角度不变
;                                            ; Straight - 发射方向受此影响
;                                            ; Bombard - 发射方向受此影响
;                                            ; Missile - `Trajectory.OffsetCoord` 和 `Trajectory.Missile.PreAimCoord` 都会受此影响
;                                            ; Engrave - 与其它不同，抛体的 `Trajectory.Engrave.SourceCoord` 和 `Trajectory.Engrave.TargetCoord` 中的L方向是否会随当前的 `Burst` 计数镜像翻转
;                                            ; Parabola - 发射方向受此影响
;                                            ; Tracing - 与其它不同，抛体的 `Trajectory.Tracing.CreateCoord` 中的L方向是否会随当前的 `Burst` 计数镜像翻转，若 `Trajectory.Tracing.TraceMode=Connection` 则 `Trajectory.Tracing.AttachCoord` 也会
Trajectory.AxisOfRotation=0,0,1              ; FLH坐标，请写完整，计算 `Trajectory.RotateCoord` 时的旋转轴，旋转轴会随单位朝向或目标位置到发射位置的向量旋转，长度不重要，只看方向（相反的向量也会反转旋转方向）
;                                            ; Straight - 有效
;                                            ; Bombard - 有效
;                                            ; Missile - 有效
;                                            ; Parabola - 有效
Trajectory.LeadTimeCalculate=no              ; 布尔值，抛体在瞄准时是否计算目标的提前量
;                                            ; Straight - 有效，计算目标的提前量会在开火时延迟1帧发射，同时若抛体的速度过低，则会无法生效
;                                            ; Bombard - 有效，若 `Trajectory.Bombard.NoLaunch=yes` 且 `Trajectory.Bombard.FreeFallOnTarget=yes` 则会同时影响到下降阶段的初始位置，若 `Trajectory.Bombard.NoLaunch=no` 则计算目标的提前量会在开火时延迟1帧发射，同时若抛体的速度过低，则会无法生效
;                                            ; Missile - 有效，抛体在运动过程中是否持续计算目标的提前量，且默认值为 `yes`
;                                            ; Parabola - 有效，计算目标的提前量会在开火时延迟1帧发射且增加性能消耗，同时若抛体的速度过低，则会无法生效
Trajectory.LeadTimeMaximum=0                 ; 整数型，抛射体最多能够预测目标还会继续运动多久（用于让抛射体不会飞得太远）
Trajectory.DetonationDistance=0.4            ; 浮点型，抛体距离目标多远距离就会引爆
;                                            ; Straight - 有效，当抛体 `Trajectory.Straight.PassThrough=yes` 时，则若此值为正则抛体在距离【发射者】这个值的距离的位置引爆，为负则抛体在穿过【目标点】这个值的绝对值的位置引爆，为零视为无穷远
;                                            ; Bombard - 有效，若为正数，代表抛体在距离目标多远距离内就会引爆，若为负数，代表若抛体目标可以移动则会将目标直接丢失转为攻击其所在位置的地面格子，若为零则关闭功能
;                                            ; Missile - 有效，抛体在距离目标多远距离就会引爆
;                                            ; Parabola - 有效，若为正数，代表抛体在距离目标多远距离内就会引爆，若为负数，代表若抛体目标可以移动则会将目标直接丢失转为攻击其所在位置的地面格子，若为零则关闭功能
Trajectory.TargetSnapDistance=0.5            ; 浮点型，抛体引爆时，当前目标在多近的距离内弹头会吸附过去
;                                            ; Straight - 有效
;                                            ; Bombard - 有效
;                                            ; Missile - 有效
;                                            ; Parabola - 有效
Trajectory.EarlyDetonation=no                ; 布尔值，抛体在 `Trajectory.DetonationHeight` 启用时，是否在上升段高于该高度时引爆，否则在下降段低于该高度时引爆
;                                            ; Bombard - 有效
;                                            ; Parabola - 有效
Trajectory.DetonationHeight=-1               ; 整数型，抛体位于什么高度时提前引爆，与 `Trajectory.EarlyDetonation` 组合使用，为负数时不生效
;                                            ; Bombard - 有效
;                                            ; Parabola - 有效
Trajectory.AllowFirerTurning=yes             ; 布尔值，抛体在发射后是否允许发射者的朝向大幅度变动
;                                            ; Engrave - 有效
;                                            ; Tracing - 有效
Inaccurate=no                                ; 布尔值，抛体是否启用随机散布，不同弹道类型存在不同含义
;                                            ; Straight - 抛体在目标位置的随机散布
;                                            ; Bombard - 抛体在目标位置的随机散布
;                                            ; Missile - 抛体在目标位置的随机散布
;                                            ; Engrave - 无作用
;                                            ; Parabola - 抛体在目标位置的随机散布
;                                            ; Tracing - 无作用
BallisticScatter.Min=0                       ; 浮点型，最小散布范围，每武器 `Range` 的攻击距离会使最小散布距离中心几格
BallisticScatter.Max=                        ; 浮点型，最大散布范围，每武器 `Range` 的攻击距离会使最大散布距离中心几格，默认值为 `CombatDamage` 中 `BallisticScatter` 的值
Gravity=6.0                                  ; 浮点型，抛体所受重力大小，不同弹道类型存在不同含义
;                                            ; Straight - 无作用
;                                            ; Bombard - 抛体在下落阶段所受的重力大小
;                                            ; Missile - 抛体所受的重力大小
;                                            ; Engrave - 无作用
;                                            ; Parabola - 抛体所受的重力大小
;                                            ; Tracing - 无作用
SubjectToGround=no                           ; 布尔值，开火前是否对前方直线的地形进行检查防止穿过地下，不同弹道类型存在不同含义
;                                            ; Straight - 除了是否检查外，还会控制抛体低于地面高度时是否立刻引爆

;                                            ; Missile - 仅是否检查，抛体低于地面高度时必定引爆
;                                            ; Engrave - 仅是否检查，抛体低于地面高度时不受影响
;                                            ; Parabola - 仅是否检查，抛体低于地面高度时必定引爆
;                                            ; Tracing - 仅是否检查，抛体低于地面高度时不受影响
```

 > 以下为 `Parabola` 弹道抛体的独有功能

在 `rulesmd.ini` 之中：

```ini
[SOMEPROJECTILE]                             ; `Parabola` 弹道抛体
Trajectory.Parabola.OpenFireMode=Speed       ; 发射模式枚举类，抛体的发射模式，包括 Speed/Height/Angle/SpeedAndHeight/HeightAndAngle/SpeedAndAngle
Trajectory.Parabola.ThrowHeight=600          ; 整数型，抛体的抛射高度，抛体的水平速度将因距离发生改变，具体的高度将由发射位置的高度和目标位置的高度取较大值并增加该值，不支持非正数
Trajectory.Parabola.LaunchAngle=30           ; 浮点型，抛体的发射角度，抛体的水平速度和最高高度将因距离发生改变，使用模式2或模式5时支持(-90.0,90.0)，使用模式4时支持(0.0,90.0)
Trajectory.Parabola.DetonationAngle=-90.0    ; 浮点型，抛体在当前速度方向与水平面之间的夹角小于该值时会提前引爆，当该值处在(-90.0,90.0)时生效，该值为零时（即需要抛体在最高点时引爆）会简化判断节约性能
Trajectory.Parabola.BounceTimes=0            ; 整数型，抛体的弹跳次数，抛体速度过快时使用弹跳会产生问题
Trajectory.Parabola.BounceOnTarget=land      ; 目标枚举类，抛体可以在哪些格子或单位上发生弹跳，包括 none|land|water|empty|infantry|units|buildings|all
Trajectory.Parabola.BounceOnHouses=all       ; 所属枚举类，抛体可以在哪些所属的单位上发生弹跳，包括 none|owner/self|allies/ally|team|enemies/enemy|all
Trajectory.Parabola.BounceDetonate=no        ; 布尔值，抛体是否可以在每次弹跳时额外引爆一次弹头
Trajectory.Parabola.BounceAttenuation=0.8    ; 浮点型，抛体弹跳伤害的衰减系数，即每次弹跳后下一次弹跳造成伤害会是本次伤害的多少倍
Trajectory.Parabola.BounceCoefficient=0.8    ; 浮点型，抛体弹跳的弹性系数，即每次弹跳后的速度是弹跳前速度的多少倍
```

- 选项 `Trajectory.Parabola.OpenFireMode` 可选类型：

| 类型             | 模式                       | 简述                             | 计算                                                                                     |
|:-----------------|:---------------------------|:---------------------------------|:-----------------------------------------------------------------------------------------|
| *Speed*          | 固定水平速度的自动计算模式 | 抛体的飞行时间是相对固定的       | 使用 `Trajectory.Speed` 和目标的坐标作为计算条件                                         |
| *Height*         | 固定最高高度的自动计算模式 | 抛体的引爆时间是相对固定的       | 使用 `Trajectory.Parabola.ThrowHeight` 和目标的坐标作为计算条件                          |
| *Angle*          | 固定发射角度的自动计算模式 | 抛体的发射角度是永久固定的       | 使用 `Trajectory.Parabola.LaunchAngle` 和目标的坐标作为计算条件                          |
| *SpeedAndHeight* | 固定水平速度和最高高度模式 | 轨迹将会随目标的高度发生高度变化 | 使用 `Trajectory.Speed` 和 `Trajectory.Parabola.ThrowHeight` 作为计算条件                |
| *HeightAndAngle* | 固定最高高度和发射角度模式 | 轨迹将会随目标的高度发生水平变化 | 使用 `Trajectory.Parabola.ThrowHeight` 和 `Trajectory.Parabola.LaunchAngle` 作为计算条件 |
| *SpeedAndAngle*  | 固定水平速度和发射角度模式 | 轨迹将会是永久固定的             | 使用 `Trajectory.Speed` 和 `Trajectory.Parabola.LaunchAngle` 作为计算条件                |

```{note}
- 在 `Trajectory.Parabola.OpenFireMode=Angle` 模式下，性能消耗较高，且由于可能攻击目标过高而无解，不建议使用此模式时启用 `SubjectToCliffs` 或启用 `AA` 并使用较小的 `MinimumRange`
- `Trajectory.Parabola.BounceTimes` 默认只能在地面、斜坡和大部分悬崖处反弹，弹跳功能和 `Trajectory.DetonationDistance` 并不冲突会同时生效，如需要抛体只会在弹跳次数耗尽后再爆炸，请将 `Trajectory.DetonationDistance` 设置为非正数
```

在 `rulesmd.ini` 之中：


| *RotateCW*   | 顺时针旋转模式 | 以FL方向偏移的合成量为半径，绕H轴顺时针旋转 |
| *RotateCCW*  | 逆时针旋转模式 | 以FL方向偏移的合成量为半径，绕H轴逆时针旋转 |

```{note}
- 在使用 `Trajectory=Tracing` 时，武器的 `IsLaser` 会持续连接发射位置和抛射体位置
```

- 原本游戏中，很多脚本需要小队所有成员全部抵达目的地周围Stray范围内才算作任务完成。在小队成员很多或者附近有建筑挡路的情况下，这可能导致目的地周围没有足够的位置放置所有小队成员，然后小队就会卡在这一步无法继续。
  - 现在修改了这个判定以试图解决这个问题，具体来说我们将使用所有小队成员的平均位置而非单个成员的位置来判断小队是否到达了目的地。

- 原本游戏中，小队需要小队所有成员位于平均位置附近Stray范围内才能开始行动。这会导致类似上述的问题。
  - 现在修改了这个判定以试图解决这个问题，具体来说我们将额外判断平均位置周围的单位数量，如果单位数量过多那就认为已经到达了目的地。

#### 脚本巡逻至路径点增强

  > - 暂无pr *by TaranDahl*
  > - [返回目录](#额外功能目录)

- 现在可以通过Phobos攻击和移动脚本类似的方式自动推算一个格子来代替路径点，单位会按照原有的逻辑巡逻过去，使用**小于0**的脚本参数会自动使用新逻辑，参数 `-n` 的解析方式如下（先将其去掉负号，并转换为**二进制数**得到 `ABCDDDEEEEEEEE` ，相反的，你可以通过计算器的程序员模式先将二进制数写出，再转换成十进制数）
  - `-n` 的二进制1-8位 EEEEEEEE
    > 目标类型，填 `AITargetType` 中值的索引或 `TargetType#` 的类型，具体是索引还是类型根据a4决定，具体参考Phobos攻击和移动脚本中n的写法（由于只有8位，所以没法填超过256的索引，不过256应该也够用了）
  - `-n` 的二进制9-11位 DDD
    > 目标选择优先级计算算法，0 - 距离近且高仇恨单位优先，1 - 距离远且高仇恨，2 - 距离近，3 - 距离远，4 - 更能保持作战方存活的优先（具体来说就是基地和MCV最高，其次战车工厂，然后其它工厂，再次其它 `KeepAlive` 的单位，最后是其它不 `KeepAlive` 的单位，若同一类型有多个，优先选择距离近的）
  - `-n` 的二进制第12位 C
    > 选择友军还是敌军，0 - 敌军，1 - 友军
  - `-n` 的二进制第13位 B
    > 选择目标使用 `AITargetType` 还是 `TargetType#` ，两者的区别参考Phobos已有的说明书，0 - `TargetType#` ，1 - `AITargetType`
  - `-n` 的二进制第14位 A
    > 是否需要目标可以被小队首领攻击，0 - 不需要，1 - 需要

In `aimd.ini`:

```ini
[SOMESCRIPTTYPE]                             ; 任意脚本类型
X=16,-n                                      ; 脚本
```

```{note}
- 没有仔细检查16号脚本的细节，目前可能有不小的bug，小心使用，遇到问题请反馈
```

#### 自动化建筑放置

  > - 暂无pr *by CrimRecya & TaranDahl*
  > - [返回目录](#额外功能目录)

在 `ra2md.ini` 之中：

```ini
[Phobos]                                     ; Phobos设置
AutomaticPlacingBuilding=yes                 ; 布尔值，是否默认启用自动放置建筑
AutomaticPlacingCombatBuilding=yes           ; 布尔值，是否默认启用自动放置防御建筑
```

在 `rulesmd.ini` 之中：

```ini
[General]                                    ; 全局通用设置
AutoBuilding=no                              ; 布尔值，所有建筑是否不需要手动点击图标放置，而是在完成建造后自动放置

[SOMEBUILDING]                               ; 只能是建筑
AutoBuilding=                                ; 布尔值，该建筑是否不需要手动点击图标放置，而是在完成建造后自动放置，默认值为 `General` 中 `AutoBuilding` 的值
AutoBuilding.Gap=1                           ; 整数型，建筑自动放置时需要隔最近的建筑多少格距离
```

- 拥有快捷键功能，位于 [控制] 面板，可以分别控制科技界面和防御界面的建筑能否自动放置
- 只有 `AutoBuilding=yes` 的建筑会受到快捷键影响而被禁止自动放置，而 `AutoBuilding=no` 的建筑不会因此而能够自动放置。默认开启状态

```{note}
- 启用 `AutoBuilding` 会在建筑完成时增加少量性能消耗，如果找不到地方放不下去，则会每隔一段时间再次进行尝试，且不支持激光围墙、闸门和地形
- 自动放置的科技建筑会被随机放置在集结点附近5格内，防御建筑会被随机放置在第二集结点附近5格内。具体设置方法请查看 `集结点增强`。
- 关于1个开关快捷键：
  - 开关快捷键默认名为 `Toggle Auto Building` ，可以在csf中添加名为 `TXT_AUTO_BUILD` 和 `TXT_AUTO_BUILD_DESC` 的项目来分别重写其在快捷键页面中的选项名称和详细说明
  - 开关快捷键默认名为 `Toggle Auto Building Combat` ，可以在csf中添加名为 `TXT_AUTO_BUILD_COMBAT` 和 `TXT_AUTO_BUILD_COMBAT_DESC` 的项目来分别重写其在快捷键页面中的选项名称和详细说明
```
